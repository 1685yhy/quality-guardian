# Deep Testing Guide — 深度测试指南

Quality Guardian 支持 4 级测试深度。浅层测试只发现 20% 的问题；深层测试能覆盖 80%+。

## 四级深度

### L1: 快速扫描（5-10 分钟）
**目标**: 发现阻断性问题（页面打不开、核心按钮无反应、明显错误）

每个 Guardian 检查 3-5 项，1 个 Simulator 画像，只走主流程。

**产出**: 即时问题清单（P0 only）

### L2: 标准验收（20-40 分钟）
**目标**: 完整的功能+体验验收（v1.0-v1.3 的默认级别）

每个 Guardian 检查 8-12 项，3-4 个 Simulator 画像，覆盖快乐路径+主要异常路径。

**产出**: 完整验收报告（含 P0/P1/P2）

### L3: 深度测试（1-3 小时）
**目标**: 系统性覆盖几乎所有用户场景

每个 Guardian 检查 15-25 项，4-5 个 Simulator 画像，每个画像执行 2-3 个完整用户旅程（不只是单步操作）。包含：

- ✅ L2 的所有内容
- ✅ 每个 Guardian 的系统性边界探索（见下方）
- ✅ Simulator 多轮测试（同一画像用不同路径达到同一目标）
- ✅ 状态持久化测试（关闭重开、刷新页面、切换标签页）
- ✅ 并发操作测试（快速连点、同时操作多个元素）
- ✅ 大数据量测试（长列表、大文件上传、大量输入）

**产出**: 深度验收报告 + 边界问题清单 + 性能观察

### L4: 穷尽测试（4-8 小时）
**目标**: 不留死角

在 L3 基础上增加：
- ✅ 所有 Simulator 画像 × 所有关键场景的笛卡尔积
- ✅ 多次重复测试（每个场景跑 3 次，排除偶然性）
- ✅ 极限条件测试（极慢网络、极小屏幕、极低内存）
- ✅ 安全渗透测试（XSS、SQL注入、CSRF、权限绕过）
- ✅ 无障碍深度审计（屏幕阅读器兼容性、键盘完整操作路径）

**产出**: 穷尽测试报告 + 安全审计报告 + 无障碍审计报告

## 系统性边界探索规则

当深度 ≥ L3 时，每个 Guardian 必须系统地探索以下边界。不是"遇到才测"，而是**主动构造边界条件**。

### 可达性边界
- 首页 → 每个一级页面 → 每个二级页面。画完整的导航树。
- 从每个页面出发，能否回到首页？（检查"迷路"风险）
- 搜索功能：搜不存在的东西、搜一个字符、搜特殊符号、搜空格
- 深层页面（4 层以上）：是否需要"面包屑"导航？

### 可理解性边界
- 每个按钮/链接的文字：脱离上下文还能理解吗？
- 每个表单字段：有 placeholder 或 label 吗？
- 每个错误消息：告诉用户"发生了什么"+"怎么解决"吗？
- 每个数字/状态：有单位吗？有对比基准吗？
- 专业术语：目标用户（非专业人士）能看懂吗？

### 可靠性边界
- 每个输入框：空值、超长文本(10000字)、特殊字符(`<script>alert(1)</script>`)、emoji、RTL 文字
- 每个表单：只填必填项、全填、填错格式、中途关闭浏览器再打开
- 每个支付/提交：点击后立即关闭页面、点击后断网、连续点击 5 次
- 每个删除操作：能否撤销？删除后相关数据是否一致？
- 登录态：token 过期后会发生什么？同时开两个标签页登录不同账号？

### 响应性边界
- 每个页面的首次加载时间（清除缓存）
- 每个按钮的点击反馈延迟（用 JS 测量，应 <100ms）
- 列表滚动：100 条、1000 条、10000 条时的帧率
- 图片加载：模拟 3G 网络下的加载体验
- 动画：是否有 `prefers-reduced-motion` 支持？

### 愉悦性边界
- 每个页面在不同屏幕尺寸下的视觉效果（320px / 768px / 1440px / 2560px）
- 暗色模式/亮色模式（如果支持）
- 空状态、加载状态、错误状态、极端数据状态——每个状态的视觉呈现
- 品牌一致性：颜色、字体、间距在所有页面中一致吗？
- 微交互：每个可交互元素都有 hover/active/focus 状态吗？

### 包容性边界
- Tab 键能否完成所有操作？（不碰鼠标）
- 屏幕阅读器能否理解每个元素？（检查 aria-label、alt、语义化 HTML）
- 色盲模拟：红绿色盲、蓝黄色盲下是否仍可区分状态？
- 字体缩放 200% 后布局是否正常？
- 系统动画关闭后是否仍可用？

## Simulator 深度探索模式

当深度 ≥ L3 时，Simulator 不应只是"自由浏览"。每个 Simulator 需要完成：

### 系统性探索清单
1. **快乐路径 × 1**: 以最自然的方式完成核心任务
2. **异常路径 × 1**: 故意走错路，看系统如何引导回来
3. **边界路径 × 1**: 用极端数据/极端操作测试边界
4. **重复路径 × 1**: 同一任务用不同方式完成（如搜索结果 vs 分类浏览到达同一商品）

### 多轮测试
同一画像、同一任务，在以下条件下各执行一次：
- 正常网络 (WiFi)
- 慢速网络 (3G 模拟)
- 首次使用（清除所有缓存/cookie）
- 回访使用（已有数据/登录态）

### 状态探索
- 打开页面 → 关闭浏览器 → 重新打开 → 状态是否保持？
- 填写一半表单 → 切换到其他应用 → 回来 → 内容还在吗？
- 登录 → 关闭标签页 → 新标签页打开 → 还需要重新登录吗？

## 深度等级选择

Orchestrator 根据以下规则自动选择：

| 条件 | 推荐深度 |
|------|---------|
| 首次运行、快速反馈 | L1 快速扫描 |
| PR 提交前、常规验收 | L2 标准验收 |
| 上线前最终检查、重要版本 | L3 深度测试 |
| 重构后、安全审计、无障碍合规 | L4 穷尽测试 |

用户可以用 `/quality-guardian accept --deep` 或 `/quality-guardian accept --exhaustive` 显式指定。
